TikTok Danmu Public - Config Guide

====================
中文说明
====================

这个公开版不使用：
- 授权码
- 管理员 Token
- 管理员客户端
- 服务器中转

首先选择监听模式：

- API 接口：使用填写的 Sign API Base/Key
- 本机直连：不使用已保存的自定义 Key，但 TikTokLive 仍使用默认公共签名服务
- 浏览器会话：打开 Chrome，使用非敏感浏览器状态，不传递登录 Cookie

用户需要填写的内容：

1. 直播间地址
- TikTok 直播间链接

2. Sign API Base
- 仅 API 接口模式使用
- 默认值：
  https://tiktok.eulerstream.com
- 如果你使用别的签名服务，就填写对应服务地址

3. Sign API Key
- 仅 API 接口模式使用
- 填写你自己的有效 Sign API Key
- 这不是授权码
- 这不是管理员 Token

4. 使用 Sign API Key
- 有效 Key 时开启
- 关闭后程序不会自动重新启用已保存的 Key

5. 代理
- 可选
- 如果直连没问题，可以留空
- 示例：
  http://127.0.0.1:7890
  socks5://127.0.0.1:7890

6. 代理路由模式
- direct：全部直连
- all：TikTok 和 Sign API 都走代理
- tiktok_only：只有 TikTok 走代理
- sign_only：只有 Sign API 走代理

7. 打印设置
- 选择打印机
- 设置纸张尺寸
- 根据需要调整模板、字体、画布

这个公开版固定行为：
- 仅本地监听
- 无授权激活
- 无后台管理
- 无远程永久编号同步
- 无云数据中心

配置文件路径：
- %LOCALAPPDATA%\SenNails\app_settings_public.json

典型使用步骤：
1. 填写直播间地址
2. 选择 API 接口 / 本机直连 / 浏览器会话
3. 仅 API 接口模式需要填写 Sign API Base/Key
4. 保存配置
5. 开始监听

====================
English
====================

This public build does not use:
- license code
- admin token
- admin client
- server relay

First choose a listening mode:

- API interface: uses the configured Sign API Base/Key
- Local direct: ignores a saved custom key, but TikTokLive still uses its default public signing service
- Browser session: opens Chrome, uses non-sensitive browser state, and does not forward login cookies

What the user needs to fill in:

1. Live URL
- TikTok live room URL

2. Sign API Base
- Used only in API interface mode
- Default:
  https://tiktok.eulerstream.com
- If you use another sign service, fill in that service base URL instead

3. Sign API Key
- Used only in API interface mode
- Fill in your own valid Sign API Key
- This is not a license code
- This is not an admin token

4. Use Sign API Key
- Enable when you have a valid key
- When disabled, the app will not automatically re-enable a saved key

5. Proxy
- Optional
- Leave empty if direct connection works
- Example:
  http://127.0.0.1:7890
  socks5://127.0.0.1:7890

6. Proxy Route Mode
- direct: all direct
- all: TikTok and Sign API use proxy
- tiktok_only: only TikTok uses proxy
- sign_only: only Sign API uses proxy

7. Printer Settings
- select printer
- set paper size
- adjust template, font, and canvas as needed

What is fixed in this public build:
- local listen mode only
- no license activation
- no backend management
- no remote permanent ID sync
- no cloud data center

Settings file path:
- %LOCALAPPDATA%\SenNails\app_settings_public.json

Typical usage:
1. Fill live URL
2. Select API interface / Local direct / Browser session
3. Fill Sign API Base/Key only for API interface mode
4. Save config
5. Start listening
